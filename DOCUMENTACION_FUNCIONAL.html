1. Resumen Ejecutivo

LabNumerator es un sistema web moderno diseñado para gestionar la cola de atención de pacientes en el laboratorio mediante:

  • Registro rápido mediante escaneo de código de barras
  • Validación automática con el sistema GXSalud mediante servicio SOAP
  • Visualización en tiempo real de la cola de espera
  • Gestión de múltiples sectores del laboratorio
  • Sincronización instantánea entre todas las pantallas mediante WebSockets
Beneficios Principales:
  • ✅ Reducción de tiempos de espera - Proceso de registro automatizado
  • ✅ Eliminación de errores - Validación automática con GXSalud
  • ✅ Transparencia - Los pacientes ven su posición en tiempo real
  • ✅ Eficiencia operativa - Gestión centralizada de múltiples sectores
  • ✅ Sin instalación - Aplicación web accesible desde cualquier navegador

3. Funcionalidades Principales

3.1 Registro de Pacientes

Descripción

Los pacientes registran su llegada escaneando el código de barras de su orden de laboratorio.

Proceso de Registro

  1. Paciente acerca código de barras al lector
  2. Sistema captura el código automáticamente
  3. Sistema valida con GXSalud mediante servicio SOAP
  4. Verifica que el turno esté vigente (dentro del horario permitido)
  5. Asigna número en cola del sector correspondiente
  6. Muestra confirmación en pantalla con datos del paciente

Validaciones Automáticas

  • ✅ El código existe en el sistema GXSalud
  • ✅ El turno no ha vencido (hora actual < hora final del turno)
  • ✅ El paciente no está duplicado en la cola actual
  • ✅ El sector asignado existe y está disponible

Tiempo de Proceso

Menos de 2 segundos promedio desde el escaneo hasta la confirmación

3.2 Visualización de Cola en Tiempo Real

Descripción

Pantallas públicas muestran la cola de espera actualizada en tiempo real, separada por sectores del laboratorio.

Información Mostrada

Campo Descripción
Número de Turno Número secuencial asignado al paciente
Nombre del Paciente Nombre completo según GXSalud
Sector Asignado Sector de laboratorio donde será atendido
Tiempo en Cola Minutos transcurridos desde el registro
Estado En espera, Llamado, En atención

Características Técnicas

  • Actualización automática mediante WebSockets (no requiere recargar)
  • Vista por sector o visualización global
  • Resaltado visual del próximo paciente a atender
  • Transiciones suaves para mejor experiencia visual
  • Latencia menor a 100ms en actualizaciones

  • Cambios de estado → Actualizaciones de estado en tiempo real

4. Flujos de Trabajo Detallados

4.1 Flujo Completo de Registro de Paciente

  1. Paciente llega con orden de laboratorio
    • Orden contiene código de barras visible
    • Paciente se acerca a estación de escaneo
  2. Escaneo del código de barras
    • Paciente acerca el código al lector
    • Lector captura el código automáticamente
    • Sistema muestra "Validando..." en pantalla
  3. Validación con GXSalud (automático)
    • Sistema envía código al servicio SOAP de GXSalud
    • GXSalud verifica el código en su base de datos
    • Retorna datos del paciente y del turno
  4. Verificación de condiciones
    • ¿El código existe? → Si NO: Error "Código no encontrado"
    • ¿El turno está vigente? → Si NO: Error "Turno vencido"
    • ¿Ya está en cola? → Si SÍ: Error "Ya registrado"
    • Si todas las condiciones se cumplen → Continúa
  5. Asignación a cola del sector
    • Sistema identifica el sector según datos de GXSalud
    • Asigna número secuencial en ese sector
    • Registra hora de llegada
  6. Notificación a todas las pantallas
    • Envía actualización via WebSocket
    • Pantallas de visualización agregan paciente a la lista
    • Panel de laboratorio actualiza contador
  7. Confirmación al paciente
    • Pantalla muestra: "✓ Registrado exitosamente"
    • Muestra nombre del paciente
    • Muestra sector asignado
    • Muestra número en cola
    • Instrucción: "Diríjase a la sala de espera del Sector X"
Nota: Todo el proceso toma menos de 2 segundos en condiciones normales de red.

4.2 Flujo de Atención en Laboratorio

  1. Personal ve la cola en el panel
    • Lista ordenada de pacientes en espera
    • Información completa de cada paciente
    • Tiempo de espera de cada uno
  2. Clic en "Llamar Siguiente"
    • Sistema selecciona el primer paciente de la cola
    • Cambia estado a "Llamado"
  3. Notificación automática
    • Nombre del paciente se resalta en pantallas públicas
    • Sonido opcional de notificación
    • Mensaje: "Sr(a). [NOMBRE], por favor pase al Sector [X]"
  4. Atención del paciente
    • Personal realiza extracción o procedimiento
    • Paciente permanece en estado "En atención"
  5. Finalización
    • Personal hace clic en "Completar"
    • Paciente se remueve de la cola
    • Actualización automática en todas las pantallas
    • Siguiente paciente avanza automáticamente
Situaciones Especiales:
  • Paciente no se presenta: Usar botón "Saltear" para pasar al siguiente
  • Paciente debe regresar: Usar "Volver a cola" para reposicionar
  • Urgencia médica: Usar "Priorizar" para adelantar en la cola

5. Integración con GXSalud

5.1 Descripción del Servicio SOAP

LabNumerator se integra con el sistema hospitalario GXSalud mediante un Web Service SOAP que provee información validada de pacientes y turnos.

Característica Detalle
Protocolo SOAP 1.1 sobre HTTP
Método labwbs01.Execute
Entrada Código de barras (10 dígitos)
Salida XML con datos del paciente y turno
Autenticación HTTP Basic Authentication
Timeout 10 segundos
Tiempo típico de respuesta 200-500 milisegundos

5.2 Datos Obtenidos de GXSalud

Campo Descripción Uso en LabNumerator
Nombre Nombre completo del paciente Mostrado en todas las pantallas
Cédula Documento de identidad Identificación única
Sector Código del sector de laboratorio Asignación automática a cola
Descripción de Sector Nombre del sector (ej: "SECTOR A") Mostrado al paciente
Fecha Fecha de la extracción programada Verificación de vigencia
Hora Inicial Inicio del turno Validación de horario
Hora Final Fin del turno Validación de no vencimiento

5.3 Manejo de Errores y Validaciones

Situación Detección Mensaje al Usuario
Código no existe GXSalud retorna Error=S "El código ingresado no se encuentra en el sistema. Verifique con personal administrativo."
Turno vencido Hora actual > Hora Final "Su turno ha vencido. El horario de atención finalizó a las [HORA]. Por favor, solicite un nuevo turno."
Ya registrado Código ya en cola "Este código ya está registrado en la cola. Por favor, espere su turno."
Error de autenticación HTTP 401 "Error de sistema. Contacte al personal de laboratorio."
Servicio no disponible Timeout o conexión rechazada "El sistema no está disponible temporalmente. Intente nuevamente en unos momentos."

6. Guía de Usuario por Rol

6.1 Para Pacientes - Estación de Escaneo

Instrucciones de Uso

  1. Llegue con su orden de laboratorio
    • Asegúrese de tener el código de barras visible y sin arrugas
    • Si está doblado, extiéndalo para mejor lectura
  2. Acerque el código de barras al lector
    • Mantenga el papel firme frente al lector
    • Distancia recomendada: 5-10 cm
    • Espere el sonido "beep" de confirmación
  3. Verifique sus datos en pantalla
    • Confirme que su nombre sea correcto
    • Note el sector asignado
    • Observe su número en cola
  4. Diríjase a la sala de espera indicada
    • Busque las pantallas del sector asignado
    • Observe su número en la lista
    • Espere a ser llamado

Si el lector no funciona (Entrada Manual):

  1. Toque el botón "Ingresar manualmente" en la pantalla
  2. Escriba el número del código de barras con el teclado en pantalla
  3. Presione "Validar"
  4. El sistema procesará de la misma forma

6.2 Para Personal de Laboratorio - Panel de Control

Inicio de Jornada

  1. Acceda al sistema desde su computadora
  2. Abra la URL: http://172.20.50.136:3000/lab
  3. Seleccione su sector de trabajo en las pestañas superiores

Durante la Atención

Para llamar al siguiente paciente:

  1. Verifique que el box esté disponible
  2. Haga clic en el botón "Llamar Siguiente"
  3. El nombre del paciente aparecerá destacado
  4. Las pantallas públicas mostrarán el llamado
  5. Reciba al paciente cuando ingrese

Para marcar atención completada:

  1. Finalice el procedimiento con el paciente
  2. Haga clic en "Completar" junto al nombre del paciente
  3. El paciente saldrá automáticamente de la cola
  4. El siguiente estará listo para ser llamado

Si el paciente no se presenta:

  1. Espere 2-3 minutos
  2. Haga clic en "Saltear"
  3. El paciente volverá al final de la cola
  4. El siguiente avanzará automáticamente

Para casos de urgencia:

  1. Identifique al paciente en la lista
  2. Haga clic en "Priorizar" junto a su nombre
  3. El paciente pasará al frente de la cola
  4. Registre la razón si es requerido

6.3 Para Operadores - Asistencia en Escaneo

Funciones del Operador

Asistencia a pacientes:

  • Ayudar a pacientes con dificultad para escanear
  • Verificar que el código sea legible
  • Usar entrada manual si el lector falla
  • Explicar a qué sector dirigirse

Resolución de problemas comunes:

  • Código dañado: Intentar entrada manual
  • Paciente sin código: Derivar a administración
  • Error "Ya registrado": Verificar si ya está en cola, indicar que espere
  • Error "Turno vencido": Derivar a administración para nuevo turno
